Welcome to the **RealCentric Generator-Agnostic Deepfake Detection** repository (AICTE Batch 7)!

This robust, multi-faceted deepfake detection system is designed to be **Generator-Agnostic**. By combining multiple feature domains—Statistical, Frequency, Wavelets, and Deep Semantic Representations (CLIP ViT-B/32)—this pipeline excels at identifying "zero-shot" or unseen deepfakes generated by novel models (e.g., GANs, Stable Diffusion) using various modeling paradigms.

---

## 🚀 Key Features

*   **Multi-Domain Feature Fusion:**
    *   **Statistical:** Mean, variance, entropy, local Mahalanobis distance.
    *   **Frequency:** Spectral slope, energy ratios.
    *   **Wavelet (Haar):** High-frequency artifact analysis.
    *   **CNN Semantic Models:** CLIP (ViT-B/32) embeddings for robust spatial extraction.
*   **Three Modeling Capabilities:**
    *   **Supervised Learning:** MLP combined with CLIP embeddings for known-distribution deepfake detection.
    *   **Unsupervised One-Class Ensemble:** PCA + Isolation Forest ensemble for zero-shot detection, trained strictly on *real* face distributions.
    *   **Autoencoder:** Out-of-distribution reconstruction anomaly detection.
*   **Media Robustness Evaluation:** Built and validated against social media degradation pipelines (JPEG compression, Gaussian Blur, aggressive downscaling).

## ⚙️ Installation

**Prerequisites:** Python 3.8+, PyTorch, OpenCV, PyWavelets, scikit-learn.

1.  Clone this repository:
    ```bash
    git clone <repository-url>
    cd deepfake_detection
    ```
2.  Install required dependencies:
    ```bash
    pip install -r requirements.txt
    ```

## 💻 Usage & Pipeline Execution

### 1. Configure the Environment
Adjust your dataset splits, feature extraction toggles (statistical, wavelets, CLIP), and hyperparameters within `config/config.yaml`.

### 2. Run the Notebook Pipeline
The Jupyter Notebooks carefully walk through the lifecycle of the system:
*   **`01` - `02`:** Pre-processing (brightness/blur filters) and extraction of tensors.
*   **`03` - `05`, `09`, `12`:** Model Training (Supervised, Unsupervised, Autoencoder, Multi-dataset, and Blur Robustness).
*   **`06`, `08`, `10`, `11`:** Benchmarking against media degradation and extensive ablation studies.

### 3. Inference (`07_inference.ipynb`)
For end-to-end testing of custom images, `07_inference.ipynb` loads the `mlp_multi_best.pt` and `ensemble.pkl` checkpoints simultaneously to verify images and produce anomaly heatmaps pinpointing unnatural artifacts.
