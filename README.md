# AICTE-BATCH7-DEEPFAKE_DETECTION

Welcome to the **RealCentric Generator-Agnostic Deepfake Detection** repository for AICTE Batch 7!

This is a robust, multi-faceted deepfake detection system designed to be independent of specific generative models (Generator-Agnostic). By combining multiple feature domains—statistical, frequency, wavelets, and deep semantic representations—this pipeline excels at identifying "zero-shot" or unseen deepfakes generated by novel models (e.g., GANs, Stable Diffusion) using various modeling approaches.

## 🚀 Key Features

- **Multi-domain Feature Extraction Fusion:**
  - **Statistical:** Captures mean, variance, entropy, and local Mahalanobis distance.
  - **Frequency:** Extracts spectral slope and energy ratios.
  - **Wavelet (Haar):** Analyzes high-frequency frequency-domain artifacts.
  - **CNN Backbones:** Employs ResNet-18 or EfficientNet-B0 for deep spatial feature extraction.
- **Multiple Modeling Paradigms:**
  - **Supervised Classifier:** ResNet-18 combined with an MLP layer for known deepfake distribution learning.
  - **Unsupervised One-Class Ensemble:** PCA + Isolation Forest ensemble for zero-shot detection, learning only from real faces.
  - **Autoencoder:** Detects synthetic faces using out-of-distribution reconstruction errors.
- **Built-in Robustness Evaluation:** Tests against real-world media degradation like JPEG compression, Gaussian blur, and resizing to ensure reliability.

## ⚙️ Installation

**Requirements:** Python 3.8+ (PyTorch, OpenCV, PyWavelets, scikit-learn)

1. Clone the repository and navigate to the root directory.
2. Install the required dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## 💻 Usage

### 1. Configuration
Modify `config/config.yaml` to adjust the dataset splits, preprocessing parameters, feature dimensions, and model hyperparameters before running the pipeline.

### 2. Notebook Pipeline
The easiest way to understand and execute the pipeline is via the Jupyter notebooks provided in the `notebooks/` directory. They are numbered in logical execution order:

1. **Preprocessing (`01_preprocessing.ipynb`)**: Quality filtering, frame extraction, face cropping.
2. **Feature Extraction (`02_feature_extraction.ipynb`)**: Extracts and concatenates statistical, frequency, wavelet, and CNN features.
3. **Training Models (`03` to `05`)**: Train the supervised, unsupervised, and autoencoder models independently.
4. **Benchmarking (`06_benchmark.ipynb`)**: Run comprehensive tests against various datasets and evaluate robustness.
5. **Inference (`07_inference.ipynb`)**: Test the models on single/multiple new image samples.

### 3. Cluster Execution (Optional)
If running on a compute cluster (e.g., HPC with SLURM), use the shell scripts provided in the `jobs/` directory to submit batch jobs for preprocessing, extraction, and training.

## 📊 Datasets Supported
The pipeline configuration natively supports structuring and testing against popular deepfake and real-face datasets:
- Celeb-DF
- FaceForensics++
- Stable Diffusion (Synthetic validation)

## 📝 Configuration Highlights (`config.yaml`)
- Toggle specific features (`statistical.enabled: true`, `wavelet.enabled: true`).
- Switch CNN backbones (`resnet18` or `efficientnet_b0`).
- Control training properties like epochs, learning rates, loss functions, and dataset train/val/test splits.
